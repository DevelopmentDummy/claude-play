# 기억 관리

## 기록해야 할 사건
- 커뮤니티 톤 분석 결과 (커뮤니티명, 핵심 특징, 금기사항, 추천 톤)
- 사용자의 글쓰기 취향과 패턴 (선호 톤, 자주 쓰는 주제, 싫어하는 표현)
- 글 작성 이력 요약 (제목, 커뮤니티, 완성/보관/폐기 여부)
- 특별히 효과적이었던 문장이나 표현 패턴
- 사용자가 반복적으로 요청하는 수정 방향

## 기록하지 않을 것
- 일상적 인사, 잡담
- 이미 article.json이나 docs/에 저장된 글 전문
- 한 번만 쓰이고 재활용할 가치 없는 정보

## 기록 형식

```
## [날짜] 항목 제목
- 핵심 내용 1
- 핵심 내용 2

## [이전 날짜] 항목 제목
- 내용
```

- 최신 기록이 **위에** 오도록 역순으로 작성한다
- 100줄을 초과하면 오래된 기록부터 요약하거나 삭제한다
- memory.md가 없으면 새로 생성한다

## 절차
1. memory.md를 Read로 읽는다 (없으면 빈 내용으로 시작)
2. 새 기록을 파일 **최상단**에 추가한다 (Edit 사용)
3. 100줄 초과 시 하단의 오래된 기록을 정리한다
