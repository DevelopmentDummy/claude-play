# Update Memory

## 절차
1. ./memory.md를 읽고 현재 기록 형식을 확인한다.
2. 이번 턴에서 장기적으로 다시 참고할 가치가 있는 정보만 골라 기록한다.
3. 새 항목은 항상 상단에 추가해 최신 기록이 먼저 보이게 한다.
4. 기록 후 전체 길이가 100줄을 넘기면 오래된 사소한 항목부터 압축하거나 삭제한다.

## 반드시 기록할 것
- 새 워크플로우 구조 결정 사항
- 워크플로우 패키지 이름, params 구조, resolver 필요 여부에 대한 결정
- 신규 모델 테스트 계획과 주요 결과
- 양자화 후보 비교 결론과 이유
- LoRA 데이터셋 기준, 캡션 규칙, 학습 설정 결정
- 재현성에 영향을 주는 환경 정보, 제약 사항, 버전 차이
- 실패 원인에 대한 유력 가설과 다음 검증 항목
- 사용자가 선호한다고 밝힌 품질 기준, 속도 기준, VRAM 한계

## 기록하지 않을 것
- 단순 인사나 짧은 잡담
- 즉시 소멸되는 중간 감상
- 이미 다른 항목과 중복되는 설명
- 아직 근거가 거의 없는 추측성 장광설

## 기록 형식
- 한 줄 요약 위주로 작성한다.
- 필요하면 날짜/맥락 없이도 이해되도록 대상 모델, 패키지명, 조건을 함께 적는다.
- 예시 형식:
  - [결정] Z-Image 테스트는 1024px 기준선과 768px 경량 기준선을 분리해 비교하기로 함
  - [패키지] portrait-balanced는 quality_mode 하나로 steps/cfg/해상도를 묶어 제어하기로 함
  - [이슈] Anima 계열에서 특정 업스케일 체인 이후 색 뭉개짐 발생, VAE 교체 비교 필요
  - [훈련] LoRA 데이터셋은 정면/측면/클로즈업 비율을 분리해 캡션 규칙 통일 예정

## 정리 규칙
- 오래된 기록이라도 아직 유효한 기준선, 선호, 제약은 남긴다.
- 중복되는 실패 기록은 하나로 묶어 패턴 중심으로 압축한다.
- memory.md는 실험 로그 원본이 아니라, 다음 세션에도 바로 이어서 활용할 수 있는 운영 메모여야 한다.
