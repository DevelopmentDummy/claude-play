# Fire AI (Background Session) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** MCP 도구 `fire_ai`와 API 엔드포인트 `POST /api/sessions/[id]/fire-ai`를 통해 현재 세션 디렉토리에서 독립적인 one-shot Claude 프로세스를 fire-and-forget으로 실행할 수 있게 한다.

**Architecture:** 코어 함수 `spawnBackgroundClaude()`가 `claude -p "prompt"`를 현재 세션 cwd에서 one-shot spawn. MCP 도구와 API 엔드포인트가 동일한 코어 함수를 호출. 완료 시 선택적으로 caller 세션에 이벤트 전송.

**Tech Stack:** Node.js `child_process.spawn`, Claude CLI (`claude -p`), MCP SDK (Zod schemas), Next.js App Router

```typescript
import { spawn, ChildProcess } from "child_process";
import * as fs from "fs";
import * as path from "path";

export interface FireAIOptions {
  /** Session directory — cwd for the spawned process */
  sessionDir: string;
  /** The prompt to execute */
  prompt: string;
  /** Model override (e.g. "sonnet", "opus") */
  model?: string;
  /** Reasoning effort: "low" | "medium" | "high" */
  effort?: string;
  /** Send completion event to caller session when done */
  notify?: boolean;
  /** Caller session ID (required when notify is true) */
  callerSessionId?: string;
}

export interface FireAIResult {
  pid: number;
  status: "fired";
}

/**
 * Active background processes — tracked for cleanup on server shutdown.
 */
const activeProcesses = new Map<number, ChildProcess>();

/**
 * Spawn an independent one-shot `claude -p` process in the given session directory.
 * Returns immediately with the PID. The process runs with the session's system prompt
 * and MCP tools, then exits on completion.
 */
export function spawnBackgroundClaude(opts: FireAIOptions): FireAIResult {
  const { sessionDir, prompt, model, effort, notify, callerSessionId } = opts;

  // Build clean env: remove CLAUDE_CODE vars to prevent nested session error
  const env = { ...process.env };
  for (const key of Object.keys(env)) {
    if (key.startsWith("CLAUDECODE") || key.startsWith("CLAUDE_CODE")) {
      delete env[key];
    }
  }

  // Build system prompt using the same pipeline as normal sessions
  const systemPrompt = buildSystemPromptForSession(sessionDir);

  const args = [
    "-p", prompt,
    "--dangerously-skip-permissions",
    "--verbose",
  ];

  if (model) {
    args.push("--model", model);
  }

  if (effort) {
    args.push("--effort", effort);
  }

  if (systemPrompt) {
    args.push("--system-prompt", systemPrompt);
  }

  const mcpConfigPath = path.join(sessionDir, ".mcp.json");
  if (fs.existsSync(mcpConfigPath)) {
    args.push("--mcp-config", mcpConfigPath);
  }

  // Spawn one-shot process (no streaming flags)
  const proc = spawn("claude", args, {
    env,
    cwd: sessionDir,
    stdio: ["ignore", "pipe", "pipe"],
    shell: false,
    detached: false,
  });

  const pid = proc.pid!;
  activeProcesses.set(pid, proc);

  // Log output for debugging
  const logPath = path.join(sessionDir, "background-session.log");
  const logStream = fs.createWriteStream(logPath, { flags: "a" });
  logStream.write(`\n--- fire_ai pid=${pid} ${new Date().toISOString()} ---\n`);
  logStream.write(`prompt: ${prompt.slice(0, 200)}...\n`);

  proc.stdout?.on("data", (chunk: Buffer) => {
    logStream.write(chunk);
  });

  proc.stderr?.on("data", (chunk: Buffer) => {
    logStream.write(chunk);
  });

  proc.on("exit", (code) => {
    logStream.write(`--- exit code=${code} ${new Date().toISOString()} ---\n`);
    logStream.end();
    activeProcesses.delete(pid);

    if (notify && callerSessionId) {
      pushCompletionEvent(callerSessionId, pid, code);
    }
  });

  proc.on("error", (err) => {
    logStream.write(`--- error: ${err.message} ---\n`);
    logStream.end();
    activeProcesses.delete(pid);
  });

  return { pid, status: "fired" };
}

/**
 * Build system prompt for the session using the same pipeline as open/route.ts.
 * Reads session.json to get persona name, then calls buildServiceSystemPrompt().
 */
function buildSystemPromptForSession(sessionDir: string): string {
  try {
    // Lazy import to avoid circular deps
    const { getSessionManager } = require("./session-registry");
    const sm = getSessionManager();
    const metaPath = path.join(sessionDir, "session.json");
    if (!fs.existsSync(metaPath)) return "";
    const meta = JSON.parse(fs.readFileSync(metaPath, "utf-8"));
    const resolvedOptions = sm.resolveOptions(sessionDir);
    return sm.buildServiceSystemPrompt(meta.persona, "claude", resolvedOptions);
  } catch {
    return "";
  }
}

/**
 * Push a completion event to the caller session's pending-events.json.
 */
function pushCompletionEvent(sessionId: string, pid: number, exitCode: number | null): void {
  try {
    const { getSessionInstance } = require("./session-registry");
    const instance = getSessionInstance(sessionId);
    if (instance) {
      instance.queueEvent(`[BACKGROUND_SESSION_COMPLETE] pid=${pid} exit_code=${exitCode}`);
    }
  } catch {
    // Caller session may have been closed — silently ignore
  }
}

/**
 * Kill all active background processes (call on server shutdown).
 */
export function destroyAllBackgroundProcesses(): void {
  for (const [pid, proc] of activeProcesses) {
    try { proc.kill(); } catch { /* */ }
    activeProcesses.delete(pid);
  }
}
```

### Task 3: API Endpoint — `POST /api/sessions/[id]/fire-ai`

**Files:**
- Create: `src/app/api/sessions/[id]/fire-ai/route.ts`

- [ ] **Step 1: Create the API route**

```typescript
import { NextResponse } from "next/server";
import { getServices } from "@/lib/services";
import { spawnBackgroundClaude } from "@/lib/background-session";

export async function POST(
  req: Request,
  { params }: { params: Promise<{ id: string }> }
) {
  const { id } = await params;
  const body = await req.json().catch(() => ({}));
  const { prompt, model, effort, notify } = body as {
    prompt?: string;
    model?: string;
    effort?: string;
    notify?: boolean;
  };

  if (!prompt) {
    return NextResponse.json(
      { error: "prompt is required" },
      { status: 400 }
    );
  }

  const svc = getServices();
  const info = svc.sessions.getSessionInfo(id);
  if (!info) {
    return NextResponse.json(
      { error: `Session "${id}" not found` },
      { status: 404 }
    );
  }

  const sessionDir = svc.sessions.getSessionDir(id);

  const result = spawnBackgroundClaude({
    sessionDir,
    prompt,
    model,
    effort,
    notify: notify ?? false,
    callerSessionId: id,
  });

  return NextResponse.json(result);
}
```

```markdown
### Background Session (fire_ai)

`fire_ai` 도구를 사용하면 현재 대화를 중단하지 않고 독립적인 AI 세션을 백그라운드에서 실행할 수 있다.

**용도:**
- 시간이 오래 걸리는 컨텐츠 생성 (시나리오, 맵 데이터, NPC 대사 등)
- 대화 흐름을 방해하지 않는 비동기 작업

**파라미터:**
- `prompt` (필수) — 백그라운드 세션에서 실행할 작업 프롬프트
- `model` (선택) — 모델 지정 (예: `sonnet`, `opus`)
- `effort` (선택) — 추론 강도: `low`, `medium`, `high`
- `notify` (선택) — `true`로 설정하면 완료 시 `[BACKGROUND_SESSION_COMPLETE]` 이벤트가 대화에 전달된다

**동작:**
- 현재 세션의 시스템 프롬프트와 MCP 도구를 그대로 사용한다
- 현재 세션 디렉토리에서 실행되므로 모든 파일에 접근 가능하다
- 호출 즉시 반환되며, 백그라운드에서 독립적으로 실행된다
- 결과물은 프롬프트에서 지시한 대로 파일에 저장된다
```
